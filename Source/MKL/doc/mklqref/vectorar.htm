Vector Arguments

Vector arguments in BLAS. Vector arguments are passed using three paramters: the vector length n, increment incx, and one-dimensional array x with the vector components stored in the following array elements:
x(1), x(1+abs(incx)), ..., x(1+(n-1)*abs(incx))

If incx is positive, then the elements in array x are stored in increasing order. If incx is negative, the elements in array x are stored in decreasing order with the first element defined as x(1+(n-1)*abs(incx)). If incx is zero, then all elements of the vector have the same value, x(1).

The dimension of the one-dimensional array that stores the vector must always be at least 1+(n-1)*abs(incx)

Vector arguments in Sparse BLAS. Sparse BLAS routines handle two types of vectors: full-storage vectors and compressed sparse vectors. Full-storage vectors are arrays that store all vector elements contiguously: a(1),a(2),...,a(n). (This corresponds to increment incx=1 in BLAS routines.)

If we assume that there are only nz non-zero elements in the vector:

a(k1), a(k2),..., a(knz)

then this vector can also be passed to Sparse BLAS in compressed form using two Fortran arrays: x (values) and indx (indices). Each of these arrays has nz elements:

x(1)=a(k1), x(2)=a(k2),..., x(nz)=a(knz);
indx(1)= k1, indx(2)= k2,..., indx(nz)= knz.
